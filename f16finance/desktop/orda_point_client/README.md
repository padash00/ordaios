# Orda Control Point Client

Первый универсальный desktop-клиент для точек.

Что уже умеет:

- при запуске открывать только форму входа
- не показывать оператору device token и настройки терминала
- пускать в настройки только через вход `super-admin`
- загружать список терминалов для super-admin через сайт
- локально привязывать программу к нужной точке только из вкладки super-admin
- после входа оператора открывать единый рабочий shell точки
- показывать калькулятор смены для `Arena`-сценария
- показывать `Ramen`-сканер товаров и быстрые долги, если устройству включён `debt_report`
- показывать отдельный долговой модуль для ручного ввода и удаления долгов
- давать super-admin управление каталогом товаров текущей точки прямо в программе
- давать super-admin сводки точки: склад, долги сотрудников, долги клиентов и историю смен
- загружать точку и список операторов через `/api/point/bootstrap`
- проверять вход оператора через `/api/point/login`
- проверять вход super-admin через `/api/point/admin-login`
- загружать устройства для super-admin через `/api/point/admin-devices`
- загружать каталог товаров точки через `/api/point/products`
- загружать отчётные сводки точки через `/api/point/reports`
- отправлять сменный отчёт через `/api/point/shift-report`
- загружать и отправлять долги через `/api/point/debts`
- сохранять отчёты в локальную offline-очередь при ошибке сети
- сохранять долговые действия в отдельную offline-очередь
- повторно отправлять накопленную очередь

Что это заменяет:

- прямой доступ из точек в Supabase REST
- жёстко зашитые `company_code`, `Supabase key` и `Telegram token` в программе

Что дальше:

1. добить полный scanner/product flow из старого `Ramen`
2. перенести оставшиеся точечные сценарии в feature-driven модули
3. включать/выключать модули по `feature_flags`
4. собрать `.exe` из одного клиента для всех точек

Сценарий работы:

1. При запуске открывается только форма входа.
2. Если терминал ещё не привязан, обычный оператор войти не сможет.
3. Super-admin входит своим логином и паролем от сайта и во вкладке `Терминал` выбирает нужное устройство точки.
4. Если это `Ramen`-точка, super-admin может во вкладке `Товары` наполнить каталог.
5. После локальной привязки терминала оператор входит своими данными от сайта.
6. Программа сама открывает нужные вкладки по точке и `feature_flags`.

Запуск:

```bash
python main.py
```

Ожидается, что на сайте уже выполнена миграция:

- `supabase/migrations/20260315_point_devices_and_bootstrap.sql`
- `supabase/migrations/20260315_point_debt_items.sql`
- `supabase/migrations/20260315_point_products.sql`

И в таблице `point_devices` создано устройство с:

- `company_id`
- `name`
- `device_token`
- `point_mode`
- `feature_flags`
